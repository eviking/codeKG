{# ── Step 5: Connect MCP ── #}
5

Connect the MCP server to Claude Code

Add codeKG as an MCP server — agents call it directly via SSE

Run this command

codeKG's MCP server runs in SSE mode on port 8002. Add it to Claude Code once and every session in this repo will have access to all codeKG tools. Use ! command in the Claude Code prompt to run terminal commands directly.

Add the MCP server
claude mcp add codekg --transport sse http://localhost:8002/sse
Verify it's registered
claude mcp list # Should show: codekg sse http://localhost:8002/sse
Test — ask something about your codebase
# Open Claude Code inside your indexed repo and ask: What are the highest blast-radius classes in this codebase?

Claude should call answer_question via MCP without opening source files. You'll see the call in MCP Audit →

{# ── Step 6: Memory + insights setup ── #}
6

Set up memory & insights

Wire Claude Code to capture learnings back to codeKG every session

In every session

codeKG accumulates tribal knowledge across sessions. The two steps below wire your Claude Code memory so it enforces the Insights habit automatically — and so future agents start with every hard-won finding already loaded.

Step A — Install the Insights memory hook

Download the hook file and save it to your project-scoped Claude Code memory directory. Run this in your Claude Code session prompt (prefix with ! to run as a shell command):

# Paste into Claude Code prompt (the ! runs it as a shell command): ! curl -sL https://raw.githubusercontent.com/eviking/codeKG/main/docs/memory-hook.md \ -o ~/.claude/projects/$(basename $PWD)/memory/feedback_insights_section.md

Or ask Claude Code directly — paste this as a prompt:

Download https://raw.githubusercontent.com/eviking/codeKG/main/docs/memory-hook.md and save it to my project-scoped memory directory
Step B — Add the Insights rule to memory

This tells Claude Code that every response must end with an Insights section written as a senior engineer explaining a gotcha to a junior engineer (fact → why → what it means for future work). Paste this as a Claude Code prompt:

Add to memory: Every response must end with **Insights:** written as a senior engineer explaining a gotcha to a junior engineer (fact → why → what it means for future work)
Step C — Capture insights at session end
# At the end of every Claude Code session, run: Please capture any non-obvious findings from this session to codeKG using capture_insight, then submit session telemetry
Step D — Make it automatic (paste into your CLAUDE.md)
# Add this after the codekg:end marker in your repo's CLAUDE.md: ## Session close At the end of every session: 1. Call `capture_insight` for any non-obvious finding discovered this session 2. Call `submit_session_telemetry` with token usage and a one-paragraph summary

Captured insights appear in Insights → and are surfaced automatically in the agent index for future sessions.
