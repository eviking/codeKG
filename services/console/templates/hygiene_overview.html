How scores are calculated

Each class is scored 0–100 across four equally-weighted dimensions: class size (method count), blast radius (transitive dependents), coupling, and documentation coverage. The repo score is the mean across all non-test classes, with a penalty for god classes above 1% of total. Token overhead estimates how many extra tokens an AI agent consumes working in this codebase versus a clean baseline — based on a multiplier of 1× (score A) up to 15× (score F) applied per class.
